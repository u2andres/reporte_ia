Descripción del proyecto
▼
ℹ️ WSAD "Prueba Asistida con IA" es un sistema de gestión escolar desarrollado para la Dirección de Educación del Gobierno de la Ciudad de Buenos Aires. Permite administrar la Planta Orgánica Funcional (POF) de los establecimientos educativos, incluyendo pedidos de cargos, reportes SQL dinámicos, valorización y equivalencias de cargos.

Cómo iniciar los servicios

1
Verificar que Herd esté corriendo — ícono en barra de tareas con puntos verdes. Las APIs wsad-auth-api y wsad-common-api deben estar con herd link + herd isolate php@8.3.
2
Iniciar Frontend v2 desde cmd.exe (NO PowerShell): 
cd "D:\prueba desarrollo\GP_0.1\GP_0.1\wsad-frontend-obelisco-v2"
npm run dev
3
Abrir http://127.0.0.1:3002 · login pestaña "Cuenta Usuario" + credenciales SGE.
4
(Opcional) Frontend v1 en paralelo:
cd "D:\prueba desarrollo\GP_0.1\GP_0.1\wsad-frontend"
npm run dev
⚠️ Importante: No usar php artisan serve para las APIs. Usar únicamente Laravel Herd con PHP 8.3 isolado. PHP 8.4 es incompatible con Laravel 10 y genera errores de deprecación (código de salida 255).
4

Sub-módulo: POF / Pedidos
▼
ℹ️ Muestra la Planta Orgánica Funcional de cada establecimiento educativo con filtros avanzados y acciones en lote.

Funcionalidades implementadas

  • Toggle de filtros (checkbox Mostrar filtros)
  • Filtros: código de colegio, código de cargo, turno, período, dependencia funcional, tipo de movimiento, estado
  • Barra de acciones en lote: acción, seleccionar todos, motivo, botón Ejecutar + badge contador de seleccionados
  • Panel de establecimiento con borde izquierdo azul GCBA #274986
  • DataTable: Cargo, Turno, Acciones, Cant. Actual/Orig, N° Pedido, Cant. pedida, Valorización, Movimiento (badge color), Estado/Usuario/Fecha, Tipo, Aprobado
  • Badge MOVIMIENTO: Incremento   Reducción

JOIN correcto para Área

-- Cadena de JOINs para obtener c650_descripcion (área)
680_POF_P (p)
  JOIN 658_ESTABLECIMIENTO_POF_P (e)  ON e.c658_id     = p.c680_658_id
  JOIN 664_MODALIDAD_POF_P (mod)      ON mod.c664_id   = e.c658_664_id
  LEFT JOIN 650_AREA_POF_P (ar)       ON ar.c650_id    = mod.c664_650_id
-- ⚠️ NO usar c652_650_id (es de la tabla cargo, no lleva al área correcta)
