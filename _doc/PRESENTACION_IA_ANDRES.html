Punto de partida

Los conocimientos del Curso de IA se aplicaron directamente a un caso real: migrar un reporte PDF complejo desde el sistema legacy Symfony 1.4 a Laravel 12, usando IA como asistente en cada etapa del proceso.

El reporte elegido para las pruebas

Del panel de reportes del sistema original se eligió el primero: Listado de Planta Completa Valorizada por Área y Año. Un reporte de alta complejidad por su volumen y tiempo de generación.

⚠️
Complejidades principales del reporte: Al ser por Área incluye todos los establecimientos involucrados (pueden ser centenares). El tiempo de generación puede ser largo. Requiere procesamiento por tandas para evitar timeout.

Pasos implementados ✅
1
Mock del módulo en entorno viejo
Se inició armando una versión simulada del módulo en Symfony 1.4 para entender la lógica de negocio antes de migrar.
2
Instalación de Laravel + librería PDF
Se instaló Laravel 12 y la librería DomPDF (Barryvdh) con un controlador de prueba para validar el entorno.
3
Reporte real con datos mockeados
Se armó el reporte "Planta Completa Valorizada" con datos mockeados para validar el diseño y estructura del PDF.
4
Mapeo de tablas → modelos Eloquent
Se mapearon las tablas necesarias desde los esquemas Doctrine 1.2 a modelos Eloquent de Laravel.
5
Parametrización del reporte
Se incorporaron los parámetros de filtro: área, año y código de establecimiento opcional.
6
Librerías de barra de progreso + combinación de PDFs
Se agregaron librerías para la barra de progreso animada (7 etapas) y la generación del PDF final.
7
Módulo migrado — funcionando en prueba
El reporte "Planta Completa Valorizada" genera PDF A4 portrait con datos reales desde sdo_db en el nuevo stack.

Cómo la IA asistió la migración

03

📋 Documentación automática

La IA documentó cada paso de la migración: desde el análisis del código Symfony hasta la instrucción precisa para implementar cada regla de negocio en Laravel.

LEGACY_RULES.mdIngeniería inversa

🔄 Traducción de stack

Convirtió la lógica Symfony + Doctrine 1.2 en instrucciones Laravel + Eloquent. Mapeo de tablas, relaciones y consultas SQL complejas con JOINs múltiples.

Doctrine → EloquentPropel → Laravel

⚡ Optimización de queries

Identificó el problema de N+1 queries (~1.600 queries por reporte) y propuso la solución: query bulk para el año anterior. Resultado: ~1.600 → ~300 queries.

~1.600 → ~300 queries

🏗 Arquitectura guiada

Definió el patrón de implementación: Route → Controller → Service → Filter → Resource. Controladores thin con lógica en Services.

Route → Controller → Service

📄 Diseño del PDF

Diseñó la plantilla Blade para DomPDF: estructura A4 portrait con escudo GCBA en base64, tabla de 3 secciones por establecimiento y pie de página con paginación.

DomPDF · Blade · A4

🎯 UX de la espera

Propuso la barra de progreso animada de 7 etapas con mensajes descriptivos para dar feedback al usuario durante la generación del PDF (puede tardar varios minutos).

7 etapas · Vue 3 · PrimeVue
💡
Conclusión: La IA funcionó como par técnico en cada etapa — no como reemplazo del desarrollador, sino como acelerador del proceso: documentando, proponiendo arquitectura, optimizando queries y resolviendo problemas puntuales de implementación.
